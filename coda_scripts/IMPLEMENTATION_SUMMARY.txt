KEY TECHNICAL CHANGES
================================================================================

BEFORE:
--------
# Users had to set this manually:
export CODA_COMPONENT_TABLE=/path/to/components.txt
startCoda --file /path/to/components.txt

# startCoda did not export it, relied on external setting
# coda_conf_functions expected it to be set externally
# Startup logging was minimal

AFTER:
------
# No manual export needed:
startCoda --file /path/to/components.txt

# startCoda now does:
export CODA_COMPONENT_TABLE="$RESOLVED_FILE"
echo "INFO: Using component table: $CODA_COMPONENT_TABLE"

# Comprehensive startup logging:
************************************************************
Starting CODA Platform and Run Control GUI
************************************************************
INFO: All components have been launched
INFO: Starting CODA platform...
INFO: Platform started (PID: 12345)
INFO: Starting Run Control GUI...
INFO: Run Control GUI started (PID: 12346)
************************************************************
CODA startup sequence complete
************************************************************

================================================================================
STARTUP SEQUENCE (CORRECTED)
================================================================================

1. Validate --file argument
2. Export CODA_COMPONENT_TABLE from --file
3. Launch all components (ROC, FPGA, PEB, etc.) in xterm windows
4. If --config flag present:
   a. Wait for pedestal files to be generated
   b. Generate vme_<hostname>.cnf for each ROC
   c. Generate vtp_<hostname>.cnf for each ROC
5. Start CODA platform in background
6. Wait 2 seconds for platform initialization
7. Start rcGUI in background
8. Display completion summary

CRITICAL: Steps 5-7 happen AFTER all components are launched and AFTER
config generation completes (if --config was used).

================================================================================
TROUBLESHOOTING
================================================================================

If platform or rcGUI don't start:
  1. Check console output for "Starting CODA Platform and Run Control GUI"
  2. Look for PID messages showing platform and rcGUI started
  3. Check processes: pgrep -f org.jlab.coda.afecs
  4. Review platform and rcGUI logs for errors

If CODA_COMPONENT_TABLE errors appear:
  1. Verify you're using latest startCoda
  2. Ensure --file argument is provided
  3. Check that component file exists and is readable
